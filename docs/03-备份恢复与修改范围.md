# 备份恢复与修改范围

## 修改范围

默认 DXGI 模式只管理游戏 `Win64` 目录内的文件：

```text
winmm.dll
OptiScaler.ini
OptiScaler.log
OptiScaler\
```

当前 canonical runtime layout 只有一套：RT Panel 负责 `winmm.dll` / `OptiScaler`，DLSS Panel 负责 `nvngx.dll` / `DLSSTweaks`。

RT Panel 不写入、删除或恢复：

```text
nvngx.dll
dlsstweaks.ini
dlsstweaks.log
```

如果检测到 `nvngx.dll` + `dlsstweaks.ini`，面板会视为 DLSS Panel 已安装，只显示兼容状态，不接管这些文件。

## 不会默认修改

```text
HKLM\SYSTEM\CurrentControlSet\Enum\PCI\...\DeviceDesc
HKLM\SYSTEM\CurrentControlSet\Enum\PCI\...\FriendlyName
```

也不会写 NVIDIA App 全局设置。

## 备份位置

每次安装都会创建：

```text
<Win64>\_nte_rt_backups\<时间戳>
```

目录内包含：

```text
manifest.json
files\
```

`manifest.json` 记录安装前哪些 RT Panel 文件存在、备份位置、文件哈希、安装模式、`owner=nte-ray-tracing-panel` 和 `runtimeLayout=rt-optiscaler-winmm-v2`。

## 恢复逻辑

恢复不是简单删除当前文件，而是按 manifest 执行：

- 安装前存在的文件：从备份复制回去。
- 安装前不存在但由工具创建的文件：恢复时删除。
- 安装前存在的目录：从备份目录恢复。
- 安装前不存在的目录：恢复时移除。
- 旧版本 manifest 如果记录过 `nvngx.dll`、`dlsstweaks.ini` 或 `dlsstweaks.log`，新版本恢复时会跳过。
- 恢复前会尽量校验当前目标仍像本工具写入的版本，避免覆盖用户或另一个 panel 后来写入的文件。

这保证恢复目标是“回到那次安装前的状态”。
