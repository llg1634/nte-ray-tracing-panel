# 常见问题

## 关闭网页后工具还在运行？

是正常现象。网页只是前端 WebUI，真正运行的是 Python 后端或 `NTERayTracingPanel.exe`。关闭标签页不会结束 exe。重新打开 `http://127.0.0.1:22642`，点击“退出工具”。

## 为什么不用 ProcMon？

ProcMon 会加载内核过滤驱动，已经实测会触发《异环》的“黑客工具”检测。因此本工具不使用 ProcMon。

## 会影响整机服务吗？

默认 DXGI 模式不改全局显卡注册表，只修改游戏目录本地文件。其它进程不会因为系统显卡名被改而误判。

## 游戏无法启动怎么办？

进入“备份恢复”，选择最近备份，勾选自动关闭游戏和启动器，点击恢复。

## 光追选项没出现怎么办？

先确认 WebUI 目标显卡选择为默认推荐的 RTX 5090，再把游戏默认画质预设调到“极致”或以上。实测光追选项通常需要在这个预设档位才出现；切到自定义低端配置后，设置界面里的选项可能消失，但已经生效的实际效果可能保留。

再看 `OptiScaler.log` 是否生成。如果没有生成，说明游戏没有加载本地 `winmm.dll`。如果生成但没有效果，尝试重新安装，或在备份后测试 Full 实验模式。

## 为什么不推荐 RTX 5080M？

当前实测中，RTX 5080M profile 只能看到“全景光追”和“关”，不能正常打开“光线追踪”。RTX 4090 profile 能显示完整光追选项，当前版本则默认推荐 RTX 5090，5080M 仅保留作实验对照。

## 为什么默认推荐 RTX 5090？

当前版本默认使用 RTX 5090 profile：`SpoofedGPUName=NVIDIA GeForce RTX 5090`、`SpoofedDeviceId=0x2B85`、`DxgiVRAM=32`。RTX 4090 仍保留为已验证备用 profile。

## 杀软拦截怎么办？

这是本地文件写入工具，打包 exe 可能被误报。可以源码运行：

```bat
run.bat
```

或者自行从源码打包。
